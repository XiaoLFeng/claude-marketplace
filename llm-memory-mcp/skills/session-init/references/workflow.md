# Session Init 工作流参考

## 快速工作流

```
新对话开始 / 用户询问状态
    ↓
并行调用 plan_list + todo_list
    ↓
统计并简洁展示
    ↓
继续正常对话
```

## 输出示例

### 有活跃任务时

```
📊 项目状态概览

📋 计划: 2 个进行中
  • [plan-auth] 用户认证系统 (45%)
  • [plan-api] API 重构 (20%)

✅ 待办: 8 个待完成
  • 🔴 紧急: 1 个
  • 🟠 高优先: 2 个
  • 🟡 普通: 5 个

需要查看详情请告诉我~
```

### 无活跃任务时

```
📊 项目状态概览

✨ 当前没有进行中的计划和待办任务
可以开始新的工作了！
```

## 最佳实践

1. **保持简洁**：不要输出过多细节
2. **突出重点**：紧急任务优先展示
3. **快速响应**：不获取详情，提高速度
4. **按需深入**：用户需要时再调用其他 skill

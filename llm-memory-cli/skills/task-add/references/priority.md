# 优先级指南

## 优先级定义

| 级别 | 值 | 名称 | 说明 |
|------|-----|------|------|
| 低 | 1 | Low | 不紧急，可以稍后处理 |
| 中 | 2 | Medium | 正常优先级（默认）|
| 高 | 3 | High | 重要，尽快处理 |
| 紧急 | 4 | Urgent | 阻塞其他任务 |

## 选择优先级

### 优先级 4 (紧急)

```
场景:
- 阻塞其他任务无法继续
- 线上故障需要立即修复
- 安全漏洞
```

### 优先级 3 (高)

```
场景:
- 当前迭代必须完成
- 影响主要功能
- 多个任务依赖
```

### 优先级 2 (中) - 默认

```
场景:
- 正常开发任务
- 无特殊紧急性
```

### 优先级 1 (低)

```
场景:
- 优化改进
- 技术债务
- 可以延后
```

## CLI 使用

```bash
# 设置优先级
llm-memory todo add todo-xxx "任务" --priority 3

# 更新优先级
llm-memory todo update todo-xxx --priority 4
```
